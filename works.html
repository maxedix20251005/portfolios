香り体験サイト改善プロジェクト

作品概要 必須
どんなサービス・サイトか、ターゲットユーザーは誰か

香りづくりワークショップの予約減少を改善するため、導線設計と訴求順を見直したサイトリニューアル。トップ〜詳細〜予約までの到達率とCVRを高めるUI設計に加え、管理画面でコンテンツ更新・効果検証まで行える構成を実装。

使用技術 必須

フロントエンド：HTML / CSS / JavaScript

バックエンド：Java / Spring Boot / MyBatis

データベース：supabase / MySQL

その他：Git / Figma

担当範囲 必須
企画・デザイン・フロントエンド・バックエンドなど

企画・デザイン・フロントエンド・バックエンド・プロジェクト管理・検証・パッケージング/デプロイ

工夫した点・アピールポイント 必須

工夫した点1：AIで市場調査・競合比較・仮説整理の初稿を高速化し、「予約数＝流入×到達率×CVR」で課題を分解。改善対象を導線と訴求に絞り、短期間でも検証可能な設計にしました。

工夫した点2：AIを使ってワイヤーフレーム案・文言案・UI部品案を複数生成し、要件に沿って取捨選択。トップ→ワークショップ→予約の導線を最短化し、管理画面から更新と効果確認ができる運用性も担保しました。

苦労した点：AIの出力は速い一方で、抽象的な提案やブランド文脈とずれる案も多く、そのままでは実装品質と一貫性がぶれやすい点に苦労しました。

どう解決したか：目的・制約・評価指標を明示したプロンプトに統一し、案を「採用/保留/却下」でレビューする運用を実施。最終判断はKPI改善見込みと実装工数で行い、AI活用と品質担保を両立しました。

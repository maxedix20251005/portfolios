# サイト運用チートシート（作業指標）

## 1. このサイトの基本方針
- 目的: ポートフォリオ兼提案資料を「意思決定しやすい形」で提示する。
- 優先順位: 可読性 > 一貫性 > 装飾性。
- 変更ルール: まず `css/style.css` の共通スタイルで整え、ページ個別の例外を最小化する。

## 2. 主要編集ファイル
- コンテンツ: `docs/01-proposal.html`
- 全体スタイル: `css/style.css`
- 引き継ぎ: `handover-notes.md`
- 履歴: `work-done-by-codex.md`

## 3. タイポグラフィ基準（現行）
- ドキュメントタイトル: `.doc-header .doc-title`
- 章見出し: `.proposal-v2 .section__title`
- 小見出し: `.proposal-v2 .h3`
- 本文: `.proposal-v2 .list`, `.proposal-v2 .lead`, `.proposal-v2 .section__lead`
- 注釈: `.proposal-v2 .note`
- 表本文: `.proposal-v2 .table`

運用指針:
- 見出し階層は `h2(章) -> h3(節)` を厳守。
- サイズ変更は単体ページではなく共通クラスへ反映する。

## 4. 01-proposalで使う主要コンポーネント
- 左ライン見出し: `.section__title--bar`
- 数式カード: `.formula-*`
- ファネル図: `.funnel-*`
- 2分岐図: `.branch-*`
- Scope強調: `.scope-*`
- 観測設計カード: `.observation-*`
- Phase1強調行: `.phase1-row`
- 技術スタック未確認情報: `.doc-table td.unverified`

## 5. 章ごとの編集観点（01-proposal）
- 01: 結論先出し。意思決定に必要な要点のみ。
- 02: 事実 -> 構造 -> 目標逆算の順。
- 03: 仮説 -> 評価 -> 戦略 -> 施策を分離して見せる。
- 04: In/Out Scope と費用を明確化。
- 05: 技術スタックは確認済み/未確認の区別を明示。
- 06: 今回対象（Phase1）を視覚強調。
- 07: 01〜06の補足として参照性を担保。

## 6. チェックリスト（修正前後で必ず確認）
- [ ] 目次リンクと実アンカーが一致している
- [ ] 見出し階層が崩れていない（章内に素の `h2` が混ざっていない）
- [ ] 表/図の直前に「目的説明」がある
- [ ] 章末の次章接続文がある
- [ ] 強調（太字・色・バッジ）が過剰でない
- [ ] `docs/01-proposal.html` と `css/style.css` の変更意図が整合している

## 7. 変更時の実務ルール
- 原則: 1テーマごとに小さく変更し、見た目確認しながら進める。
- 変更記録: 大きな修正後は `work-done-by-codex.md` へ追記。
- 引き継ぎ: 再開前提の変更は `handover-notes.md` に要点を反映。

## 8. よくある崩れと対策
- 問題: 目次リンク先ズレ  
  対策: `id` と `href` を同時に確認。
- 問題: セル内改行で読みづらい  
  対策: 表をカード化して上下レイアウトへ。
- 問題: 強調対象が分かりにくい  
  対策: 行クラス（例: `phase1-row`）で限定強調。

## 9. 再開時の最短プロンプト
```txt
site-cheatsheet.md と handover-notes.md を前提に、
docs/01-proposal.html の最終校正を進めてください。
優先: リンク整合 -> 見出し階層統一 -> 表の目的1行追加 -> 章間接続文。
```

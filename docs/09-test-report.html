1 テスト計画

テスト計画の概要 必須
テストの対象範囲、入口条件、完了条件、成果物を明確に記載します。

概要: 本計画は、inim-dx の公開サイトと管理機能に対し、経験の浅いテスターでも同一品質で実行できるテスト運用を定義する。

項目 内容
対象範囲 ・公開サイト：Home、Workshop、Booking、商品、記事、検索、問い合わせ、法務ページ
・会員機能：登録、ログイン、アカウント管理
・予約導線：選択 -> 入力 -> 確認 -> 完了
・管理機能：F-001〜F-010（ログイン、導線設定、商品、記事、検索、アセット、ユーザー、予約/問い合わせ、運用設定・監査）
・外部連携：Supabase Auth / DB / Storage
入口条件 ・テスト環境URL、テストユーザー、検証データが準備済み
・最新のテストケースと実行手順書が共有済み
・対象ビルドがデプロイ済みで動作確認可能
完了条件 ・重大欠陥（S1）0件
・主要シナリオ成功率95%以上
・認証/認可/データ保護の必須項目100%実施
・NGケースは欠陥管理に登録され再試験結果まで記録済み
テスト方針 ・正常系 -> 異常系 -> 境界値の順で実施する
・機能単位に加えて業務シナリオ単位で結合確認する
・証跡（画面キャプチャ/ログ）はNGだけでなく代表OKケースも残す
成果物 ・03_テストケース：結果記録（OK/NG/保留/未実施）
・04_欠陥管理：不具合票（再現手順、期待/実際、重大度、状態）
・05_実行サマリ：件数、OK率、公開判定情報

1 モックの前提

位置づけ 必須
`05-wireframe` の汎用 CRUD 画面を、`07-specification` の業務画面へ具体化した試作です。

このページでは、「ログイン」「ダッシュボード」に加えて、`トップ編集 / 導線設定 / カード管理 / 商品管理 / 店舗管理 / 記事管理 / ユーザー管理 / 予約・問い合わせ管理 / アセット管理` の各画面を個別UIとして可視化しています。

実装前提を固めることが目的のため、見た目は本番デザインではなく、情報構造と操作導線の確認を優先したモックアップです。

誰が / 何のために使うか

店舗管理者または運用担当者が、管理画面へ安全に入るために利用します。権限に応じた操作範囲を判定する起点です。

操作手順と期待される効果
  1. メールアドレスとパスワードを入力して `ログイン` を押す。
  2. 認証が成功するとダッシュボードへ遷移し、利用可能なメニューだけが表示される。
  3. 認証に失敗した場合はエラーが表示され、不正ログインや入力ミスを早期に検知できる。

誰が / 何のために使うか

顧客対応担当者や店舗責任者が、予約・問い合わせ案件を一元管理し、対応状況を更新するために使います。

  3. 結果として、問い合わせ滞留の可視化と解消が進み、対応スピードと品質が安定する。

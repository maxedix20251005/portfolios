リレーション 種類 意味
`auth.users` → `user_profiles` 1対1 Supabase Auth の認証主体に対して、アプリ側プロフィールを1件対応させる。
`roles` ↔ `user_profiles` 多対多 `user_role_assignments` を介して、将来の複数権限付与に対応する。
`content_assets` → `top_hero_items` 1対多 1つのアセットを複数のヒーロー/掲載要素から再利用できる。
`user_profiles` → `top_hero_items` / `journey_steps` 1対多 誰が更新したかを保持し、運用履歴を追跡できる。
`user_profiles` → `reservations` / `inquiries` 1対多 1人の顧客が複数の予約・問い合わせを持てる。
`stores` → `reservations` 1対多 1店舗に複数予約が紐づく。店舗管理機能拡張時も整合する。
`reservations` / `inquiries` → 各ステータス履歴 1対多 対応状況の変更履歴を監査目的で残す。

補足: `auth.users` は Supabase 管理テーブルであり、通常の業務カラム追加は `user_profiles` 側で行う。`stores` は現時点では Could 機能だが、予約ログの参照整合性を保つため ER 図に含めている。
